## 프로젝트 개요
전기차 중고 부품 거래 플랫폼 개발을 해야한다.
웹앱 형식의 서비스가 될 것이다. 리액트, node js 기반의 aws 서버리스 프로젝트를 기획중이다. 
실제 사용할게 아니라 로그인 같은건 구현할 필요없다.
다만 시작시 중고 부품 구매/ 판매로 나누어 들어 갈 수 있게끔 해야한다.

1. 연구 필요성 및 목표
2010년대 초반 출시된 1세대 전기차의 수명이 종료되면서, 향후 사용 종료된 
전기차(EV)가 급격히 증가할 것으로 예상된다. 이들은 개발 초기 단계에서 출시
된 EV로, 제조 과정 시 정형화된 매뉴얼 없이 서로 다른 소재와 규격의 부품이 
사용된 상황이다. 이로 인해 모델별, 부품별 검증된 데이터로 구성된 DB와 체
계적인 재활용·재사용 프로세스 구축의 필요성이 대두되고 있다. 따라서 본 연구는 분산된 부품별 물성 데이터를 하나의 데이터베이스에서 통
합적으로 관리하고, 자체 개발한 알고리즘을 통해 각 기업의 공정 조건에 부합
하는 원료를 효율적으로 확보하는 것을 목표로 진행하고자 한다. 

“유사 활용 사례” 를 벡터화 하여 RAG에 활용

1. 이 부품이 어디다 쓸수 있는지 (다이렉트로 나오게)
2. 내 회사가 어떤걸 하는데, 어떤게 쓸모있을지 (AI 활용)
    1. 핵심기능
    2. 그래서 이걸(우리 앱) 어디다 써먹을 수 있을지 확실히 파악 가능
    3. 인력 보충을 덜 해도 된다
    4. 현대차는 이런게 이미 있지만 재고를 털거나 더 비싸게 팔 수도 있지 않을까?
3. 원하는 부품 등록후 등록되면 알람 
4. 데이터들을 팀원(실제 서비스에선 고객)들이 수시로 넣을 수 있게끔
    1. MCP로 합성 데이터 생성 구현 기획중
    2. 개발에 익숙치 않아도 쉽게 할 수 있도록
5. 서버리스 (온프레미스 컴퓨터조차 아깝다! → 아까운걸 떠나서 전원공급 정전등의 우려)

B2B 계약을 중개하는 플랫폼
